Lesson Video Localization

Translating a lesson into another language — voice & lips

The standard operating procedure: same teacher, same face, now speaking the target language — with three independent accuracy checks built in so it stays right across hundreds of videos.

Audience · localization team Updated 2026-06-25 Scripture per language SOP · workflows/translate_lecture.md

Probabilistic AI does the reasoning — transcription, translation, voicing. Deterministic checks verify each one. The discipline that keeps quality high at scale is simple: every step that can introduce an error is verified by a different engine than produced it, and every paid step sits behind a cheap check.

01 The pipeline at a glance

Five stages, three checkpoints. Read it top to bottom — each check (in green) clears the work before it moves to the next, more expensive stage.

1 · Transcribe sourceScribe v1 + Whisper large-v3
✓ Check transcription — reconcile the two ASRs
2 · Translatescripture → published version, tight to timing
✓ Check translation — blind round-trip + independent judge new
3 · Re-voice in the cloneElevenLabs · short chunkspaid
✓ Check voicing — transcribe the audio back
4 · Lip-sync the real footagefal sync-lipsync v2pro, conformpaid
5 · Finish & deliverQC specs, frame, color

Audio-only dub? Run stages 1–3 and the first two checks, then deliver — skip lip-sync.

02 The three checks (why this stays accurate)

Each check runs on a different engine than the step it verifies, so nothing grades its own work. This is the heart of the system.

CheckVerifies withCatches
1 · TranscriptionA second ASR (Whisper vs Scribe)Mis-heard names, numbers, scripture references
2 · TranslationA non-Claude model (Gemini) — blind back-translation + judgmentDropped / added / reversed meaning, wrong references
3 · VoicingTranscribe the generated audio backVoice-clone hallucination or drift
Important — checks build on each other The translation check sits on the same transcript the translation used, so it can't catch a transcription error. That's why Check 1 is non-negotiable: get the words right first, and the rest of the chain stays clean.

03 Stage 1 — Transcribe (dual-ASR)

1

Get the source words exactly right

This is the foundation. Every later stage inherits these words — a wrong name here becomes a wrong name in the final video. Run two transcribers and reconcile.

.venv/bin/python tools/extract_audio.py [video] --out .tmp/NAME.wav
.venv/bin/python tools/transcribe_diarize.py [video] --language [iso] --out .tmp/NAME_scribe.json
.venv/bin/python tools/transcribe_vo.py .tmp/NAME.wav --model large-v3 --out .tmp/NAME_whisper.json

Scribe (ElevenLabs scribe_v1) is primary — best accuracy, punctuation, proper nouns. Whisper large-v3 is the cross-check (run it on fal wizper for GPU speed if local is slow).

✓ Check 1 — reconcile the transcripts Diff the two. Where they disagree is where the errors hide — focus on proper nouns, numbers, and scripture references. Pick the correct reading (verify against the audio if unsure) and lock one clean source transcript before translating.

04 Stage 2 — Translate

2

Translate segment by segment

Preserve meaning, register, and timing. Write the aligned source + target into a dub-script JSON so every later step shares the same segments.

  • Scripture → the published version for the target language (table below) — never a literal back-translation. Identify the reference and drop in the canonical verse.
  • Keep theological terms and proper names correct and consistent.
  • Match spoken duration per segment — length drift hurts lip-sync. Trim filler and clean disfluencies so it reads naturally.
Target languageBible versionUse for --scripture-version
EnglishNew International VersionNIV
SpanishReina-Valera 1960RVR1960
ItalianNuova Riveduta 2006NR2006
✓

Check 2 — the translation accuracy double-check new

An independent model (Gemini — not the one that translated) does a blind round-trip back-translation and scores each segment for faithfulness. Review only what it flags, fix, re-run.

# set --scripture-version to the target language's version (NIV / RVR1960 / NR2006)
.venv/bin/python tools/check_translation.py --script .tmp/NAME_dubscript.json \
   --source-key es --target-key en --source-lang Spanish --target-lang English \
   --scripture-version NIV --model google/gemini-2.5-pro --out .tmp/NAME_check

It produces a per-segment scorecard (.md + .json) and lists only the segments to review. It knows scripture is intentionally in the published version (won't flag the wording) and that filler may be cleaned (won't flag trims).

✓ Check 2 — pass criteria No major flags outstanding. Read every flagged segment against the source, fix the translation, and re-run until only intended differences remain. This is the cheapest place to catch a meaning error — before a cent is spent on voice or video.

05 Stage 3 — Re-voice in the clone

3

Speak the translation in the teacher's own voice first paid step

Generate the target-language audio in the speaker's ElevenLabs clone, conformed to the original timing.

.venv/bin/python tools/build_podcast_dub.py --script .tmp/NAME_dubscript.json \
   --work .tmp/NAME_work --out-clean OUT_clean.mp3 --out-timed OUT_timed.mp3 --mode both \
   --split-chars 180 --stability 0.7 --similarity-boost 0.9 --speaker-boost
⚠ Cross-lingual clones drift — use short chunks When a clone speaks a language it was not trained on (e.g. a Spanish-trained teacher voiced in English), it garbles, loops, or drifts into foreign-sounding audio on long passages — even at default settings. The fix is --split-chars 180 with --stability 0.7 --similarity-boost 0.9 --speaker-boost: short inputs re-anchor the model so it can't wander. (Voicing into the clone's own native language doesn't need this.)
✓ Check 3 — transcribe the audio back Transcribe the generated audio (fal wizper and/or Scribe) and diff against the script. Two ASRs flagging garble in the same spot = real bad audio → re-voice that segment. One ASR alone gives false alarms on accented speech. Intentional repetition (e.g. "how long ×3") is fine.

06 Stage 4 — Lip-sync the real footage

4

Re-sync the actual footage to the new audio paid · ~$3/min

The standard recipe — verified to beat the alternatives on color, motion, and sync.

.venv/bin/python tools/fal_lipsync.py --video [video] --audio [revoiced].mp3 --out "OUT.mp4" \
   --conform --model fal-ai/sync-lipsync/v2 --lipsync-version lipsync-2-pro

--conform matches the voice to the video's exact length (pitch-preserved) and runs with no remap → native-speed footage, frame-locked to the source. Output is auto re-tagged to the correct color space.

⚠ Don't substitute the wrong model Never use omnihuman (it regenerates an avatar from a still — wrong tool) or Kling (60-second cap, re-times to 30fps, crops the frame). Only lip-sync on-camera face shots; slides and B-roll just take the new audio track.

07 Stage 5 — Finish & deliver

5

Quality-check and hand off

Confirm the output is frame-locked to the source (same duration, 1080p, fps), spot-check a frame, verify color tags, and deliver. Because it stays frame-locked, the After Effects mouth-only hybrid is available free on hero shots if a face shot needs perfection.

✓ Final gate — native-speaker QA The automated checks raise the floor; they don't replace a human ear. Every language gets a native speaker reviewing the translation and the final cut before it ships.

08 Non-negotiables

  • Get consent before cloning anyone's voice — it's their likeness, and ElevenLabs requires it.
  • Prove a new language or voice on a ~1-minute clip first (~$1) before committing a full lesson.
  • Never skip Check 1. A wrong word in the transcript silently becomes a wrong word in the final video.
  • Clear every major translation flag before voicing. Errors are cheapest to fix as text.
  • Scripture always comes from the published version for that language (English NIV · Spanish Reina-Valera 1960 · Italian Nuova Riveduta 2006), never a back-translation.
  • Native-speaker QA is the last word on every language.

10 Per-video checklist

  • Consent confirmed; voice clone resolves (--list-voices)
  • New language/voice? Proved on a ~1-min clip first
  • Dual-ASR transcribe (Scribe + Whisper large-v3)
  • Check 1: transcripts reconciled; names/numbers/refs verified
  • Translated; scripture in the language's published version; timing tight; disfluencies cleaned
  • Check 2: check_translation.py run; all major flags resolved
  • Re-voiced in clone (cross-lingual? --split-chars 180)
  • Check 3: transcribe-back diff clean (no shared-spot garble)
  • Lip-synced — v2pro lipsync-2-pro, conform / no-remap
  • QC: frame-locked, 1080p, color tags; spot-checked a frame
  • Native-speaker QA on translation + final cut
  • Delivered
